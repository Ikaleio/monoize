# URP v2 Flat Protocol Test Matrix Specification

## 0. Status

- Version: `1.0.0`
- Product name: Monoize
- Internal protocol name: `URP v2`
- Scope: doc driven protocol contract matrix for flat URP v2 work on downstream `/v1/responses`, `/v1/chat/completions`, and `/v1/messages`, plus Monoize internal flat URP invariants that later tests must enforce.

## 1. Purpose and precedence

PTM-1. This file is the repository local source of truth for protocol contract test coverage required by the flat URP v2 redesign.

PTM-2. Each matrix row defines one behavior that later implementation or test tasks MUST verify.

PTM-3. Each matrix row MUST cite at least one authoritative source. Valid authorities are official external protocol documents and existing Monoize specs when the row is Monoize internal rather than protocol native.

PTM-4. Each matrix row MUST name one concrete target test location. A target location is either:

1. an existing test function, or
2. a planned test function or planned subsection theme in an existing test file.

PTM-5. `Existing` means the named test already exists in the repository at the time this matrix was written. `Planned` means the named test does not yet exist and MUST be added before the dependent flat URP v2 task is considered complete.

PTM-6. This file is a testing contract, not an implementation summary. If current implementation behavior disagrees with an authoritative source cited here, the source and referenced Monoize specs win.

## 4. Execution rule for later tasks

PTM-7. A later flat URP v2 task that changes one matrix row's behavior is not complete until the row's target coverage is implemented or updated and the resulting tests validate the cited authority.

PTM-8. If a later task discovers that one row cites the wrong authority or wrong target location, that task MUST update this matrix in the same change that updates the tests.
